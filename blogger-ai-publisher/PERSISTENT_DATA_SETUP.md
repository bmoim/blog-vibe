# 영구 데이터 저장 설정

이 버전은 두 단계로 데이터를 보호합니다.

1. 기본: 현재 브라우저의 IndexedDB에 초안·주제 계획·조회 기록·버전 정보를 자동 백업합니다.
2. 권장: `DATABASE_URL`이 있으면 PostgreSQL에 데이터와 생성 이미지를 자동 동기화하고 서버 재시작 전에 복원합니다.

## Render PostgreSQL 연결

1. Render에서 PostgreSQL 데이터베이스를 생성합니다.
2. 웹서비스와 같은 Region을 선택합니다.
3. 데이터베이스의 Internal Database URL을 복사합니다.
4. Blogger AI Publisher 웹서비스의 Environment 화면을 엽니다.
5. 환경 변수를 추가합니다.

```text
Key: DATABASE_URL
Value: 복사한 Internal Database URL
```

6. Save, rebuild, and deploy를 실행합니다.
7. 배포 후 `/history.html`을 열어 `PostgreSQL 영구 저장` 상태를 확인합니다.

## 자동 저장 대상

- 저장된 블로그 초안
- 주제 발굴 계획
- 조회·작업 기록
- 성장 센터 설정
- 예약 발행 대기열
- 초안 버전 기록
- 최신성 감시 결과
- 일일 생성 사용량
- 생성 이미지 파일

Google OAuth 토큰은 별도의 암호화 자동 연결 기능으로 관리되며 일반 데이터 스냅샷에는 포함하지 않습니다.

## 작동 방식

- 서버 시작 전 PostgreSQL의 최신 스냅샷을 복원합니다.
- JSON 데이터가 변경되면 약 1.2초 후 자동 동기화합니다.
- 생성 이미지가 추가·삭제되면 파일 테이블에 자동 반영합니다.
- 5분마다 전체 데이터 스냅샷을 다시 저장합니다.
- 서버 종료 신호를 받으면 마지막 동기화를 시도합니다.
- 데이터베이스가 없거나 일시적으로 연결되지 않으면 브라우저 IndexedDB 백업을 계속 사용합니다.

## 확인

프로그램 상단의 `기록·데이터`를 열면 다음을 볼 수 있습니다.

- DB 연결 상태
- 저장된 초안 수
- 저장된 주제 계획 수
- 조회·작업 기록 수
- 버전 백업 수
- 마지막 동기화 시간

`지금 영구 저장` 버튼을 누르면 즉시 동기화합니다.
