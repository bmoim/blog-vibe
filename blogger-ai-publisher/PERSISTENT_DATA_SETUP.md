# 영구 데이터 저장 설정

이 버전은 두 단계로 데이터를 보호합니다.

1. 기본: 현재 브라우저의 IndexedDB에 초안·주제 계획·조회 결과·작업 기록·버전 정보를 자동 백업합니다.
2. 권장: `DATABASE_URL`이 있으면 PostgreSQL에 데이터와 생성 이미지를 자동 동기화하고 서버 재시작 전에 복원합니다.

## 기존 데이터가 있는 경우의 안전한 연결 순서

Render 환경 변수 추가는 재배포를 발생시킬 수 있으므로 다음 순서를 지킵니다.

1. 현재 성장 센터에서 `전체 백업 다운로드`를 먼저 실행합니다.
2. 영구 저장 기능이 포함된 최신 코드를 먼저 배포합니다.
3. 프로그램을 한 번 열어 상단에 `브라우저 자동 백업됨` 표시가 뜨는지 확인합니다.
4. 기존 초안이 보이지 않으면 성장 센터의 `백업 복원`으로 1번 파일을 복원합니다.
5. 그다음 Render PostgreSQL을 만들고 `DATABASE_URL`을 연결합니다.
6. 재배포 후 `/history.html`에서 `PostgreSQL 영구 저장` 상태를 확인합니다.
7. 마지막으로 `지금 영구 저장`을 한 번 누릅니다.

## Render PostgreSQL 연결

1. Render에서 PostgreSQL 데이터베이스를 생성합니다.
2. 웹서비스와 같은 Region을 선택합니다.
3. 데이터베이스의 Internal Database URL을 복사합니다.
4. Blogger AI Publisher 웹서비스의 Environment 화면을 엽니다.
5. 환경 변수를 추가합니다.

```text
Key: DATABASE_URL
Value: 복사한 Internal Database URL
```

6. Save, rebuild, and deploy를 실행합니다.
7. 배포 후 `/history.html`을 열어 `PostgreSQL 영구 저장` 상태를 확인합니다.

## 자동 저장 대상

- 저장된 블로그 초안
- 주제 발굴 계획과 추천 결과
- Search Console·GA4·AdSense 대시보드 조회 결과
- 최신 정보 검수·품질 검사·중복 키워드·내부링크 추천 결과
- 색인 상태와 깨진 링크 점검 결과
- 조회·작업 기록
- 성장 센터 설정
- 예약 발행 대기열
- 초안 버전 기록
- 최신성 감시 결과
- 일일 생성 사용량
- 생성 이미지 파일

Google OAuth 토큰은 별도의 암호화 자동 연결 기능으로 관리되며 일반 데이터 스냅샷에는 포함하지 않습니다.

## 작동 방식

- 서버 시작 전 PostgreSQL의 최신 스냅샷을 복원합니다.
- JSON 데이터가 변경되면 약 1.2초 후 자동 동기화합니다.
- 생성 이미지가 추가·삭제되면 파일 테이블에 자동 반영합니다.
- 5분마다 전체 데이터 스냅샷을 다시 저장합니다.
- 서버 종료 신호를 받으면 마지막 동기화를 시도합니다.
- 데이터베이스가 없거나 일시적으로 연결되지 않으면 브라우저 IndexedDB 백업을 계속 사용합니다.
- 성장 센터에서 실행한 주요 조회 결과는 최대 400건, 작업 기록은 최대 3,000건까지 저장합니다.

## 확인

프로그램 상단의 `기록·데이터`를 열면 다음을 볼 수 있습니다.

- DB 연결 상태
- 저장된 초안 수
- 저장된 주제 계획 수
- 저장된 조회·분석 결과 수
- 조회·작업 기록 수
- 버전 백업 수
- 마지막 동기화 시간
- 저장된 조회 결과의 전체 JSON 내용

`지금 영구 저장` 버튼을 누르면 서버 저장소와 브라우저 백업을 즉시 갱신합니다.
